선생산 관리 — 일련의 과정

Soltri 쏠트리 · ERP → 분석 → 생산지시 → 자동 완료
1
ERP
ERP에서 엑셀 다운로드
매일 아침 ERP 시스템에서 최신 데이터를 수동으로 다운받습니다.
라벨조회.xlsx주문 이력 전체 (거래처·치수·수량)
완제품재고창고.xlsx현재 창고 재고 현황
가공작업조회.xlsx최근 7일 생산 이력
발주조회.xlsx안전재고 발주 이력
2
BAT
start.bat 실행
배치파일 더블클릭 한 번으로 전체 프로세스가 자동 시작됩니다.
① refresh.py 실행 → ② 대시보드(index.html) 자동 오픈
3
PYTHON
refresh.py — 데이터 분석 및 업로드
엑셀 4개를 읽어 분석하고 결과를 구글시트에 업로드합니다.
주문 빈도 분석3회 이상 · 총수량 1000 이하
현재고 매핑재고율(%) 계산
생산이력 확인최근 7일 생산 중 여부
발주조회 매칭안전재고 발주일 추출
점수 산정빈도점수 + 긴급도
구글시트 업로드선생산리스트 + 주문이력
4
DASHBOARD
index.html — 대시보드 로드
구글시트에서 최신 데이터를 JSONP로 불러옵니다.
동시에 Apps Script에서 생산지시 중 / 생산지시 완료 목록도 로드합니다.
구글시트 (선생산리스트)JSONP → data 로드
Apps Script (무덤탭)생산지시 중·완료 상태 복원
autoRemoveGraveyard()발주 확인된 항목 자동 이동
autoCleanCompleted()재고 충족 항목 자동 제거
5
AUTO
자동 상태 판단 (로드 시 매회 실행)
생산지시 중 항목의 latest_balju(최근 발주일)가 생산지시 날짜 이상이면 → 생산지시 완료로 자동 이동
생산지시 완료 항목의 재고율이 필터 기준 초과 → 완전 제거
✅ 발주 확인됨 latest_balju ≥ 생산지시 날짜
→ 생산지시 중 에서
→ 생산지시 완료 로 이동
(Apps Script 자동 저장)
🔄 재고 충족 재고율 > 필터 슬라이더값
→ 생산지시 완료 에서
→ 완전 제거
(목록에서 사라짐)
⏳ 대기 중 발주 미확인 또는
재고 미충족 상태
→ 현재 섹션 유지
(다음 refresh까지 대기)
6
USER
사용자 — 생산지시 중 등록
선생산 필요 카드에서 생산지시 중 버튼 클릭
→ 해당 날짜가 Apps Script(구글시트 무덤탭)에 저장됨
→ ERP에 안전재고 발주 등록 후 발주조회 엑셀 다운로드 대기
