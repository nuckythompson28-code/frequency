## 개요

ERP에서 내려받은 엑셀 3종을 분석하여, **반복 주문되는 W/R 품목**을 자동 탐지하고
선생산 우선순위를 대시보드로 시각화한다. 발주 시 ERP 업로드용 엑셀을 원클릭 생성한다.

## 주요 로직

### refresh.py — 점수 산정
| 항목 | 배점 |
|------|------|
| 주문 빈도 (freq/20 × 50) | 최대 50점 |
| 재고율 0~5% | +50점 |
| 재고율 5~10% | +40점 |
| 재고율 10~20% | +30점 |
| 재고율 20~50% | +15점 |
| 재고율 50% 초과 | +0점 |

- **HYUNDAI CN** 주문 제외
- **안전재고** 주문번호 제외
- 재고율 30% 초과 품목은 `in_production = True` 처리

### refresh.py — 발주조회 매칭
업체치수 문자열 (`WEAR RING_PW D110D105W15A`) 에서 정규식으로 치수 파싱:
```
D([\d.]+)D([\d.]+)W([\w.]+)  →  [110, 105, 15A]
```
frozenset 기반으로 W/R 품목과 매칭 → 최근 발주일·발주수량(col[20]) 연결

### index.html — 엑셀 다운로드
SheetJS (`xlsx-0.20.3`) 사용, 3개 시트 생성:
- **Sheet1**: PROFORMA INVOICE (발주서 본문)
- **변환시트**: ERP 업로드용 22열 포맷
- **정의값**: 재질코드 조회표 (W/CN10, W/RS40, B/CN10 등)

## 실행 방법

1. ERP에서 엑셀 4종 다운로드 → `ERP_downloaded_data/` 에 저장
2. `start.bat` 더블클릭
3. 브라우저 자동 오픈 → `localhost:8585`
4. 카드 클릭으로 발주 품목 선택 → **엑셀 다운로드** 버튼
