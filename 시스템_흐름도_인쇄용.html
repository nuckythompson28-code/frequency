2. 상태 전환 규칙 (8가지)
#From → To방식트리거상세 설명
1
선생산 필요→생산지시 중
 수동카드 클릭 발주할 품목 클릭 → graveyard에 날짜 기록 → "생산지시 중"으로 이동
2
생산지시 중→생산지시 완료
 자동start.bat 재실행 발주조회.xlsx에서 안전재고 발주일 감지 → 발주일 ≥ 지시일이면 자동 이동
3
생산지시 중→생산지시 완료
 수동[완료] 버튼 ERP에 안 잡힐 때 (구두 발주 등) 마우스 올려서 [완료] 클릭
4
생산지시 중→선생산 필요
 수동카드 클릭 (복귀) 발주 취소 → 카드 클릭으로 되돌리기. Undo 버튼으로도 복구 가능
5
이미 생산중→생산지시 중
 수동카드 클릭 7일 가공이력에 잡혔지만 우리 발주 건일 수 있음 → 클릭해서 추적 시작
6
모든 상태→재고 충분
 자동재고율 > 30% 어떤 상태든 재고 기준 초과 시 자동 이동. 이력은 archived에 보존됨
7
graveyard/completed→추적 불가
 자동ITEMS에서 소실 데이터에서 빠진 품목 자동 감지. 경고 표시 + [제거] 버튼 제공
8
추적 불가→(삭제)
 수동[제거] 버튼 더 이상 필요 없으면 [제거] 클릭 → graveyard/completed에서 완전 삭제

안전장치 & 사용법 & 점수 산정 & FAQ
3. start.bat 재실행 시 안전장치 (3가지)
⚠
시나리오 1: 품목 소실
문제: 생산지시 중/완료에 넣었는데, 다음 실행 시 freq<10이 되어 data_wr.json에서 빠짐 → 카드 사라짐
대응: "추적 불가" 섹션에 경고 표시. 키/날짜 확인 후 [제거]로 정리 가능
💾
시나리오 2: 이력 소실
문제: 재고 일시 증가로 "재고 충분" 이동 → completed 삭제 → 다시 재고 떨어지면 이력 없이 처음부터
대응: completed → 재고 충분 시 archived에 이력 보존 (지시일/완료일/졸업일 기록)
🔄
시나리오 3: 중복 표시
문제: graveyard에 넣었는데 7일 지나면 in_production이 꺼져서 "선생산 필요"에 중복 등장
대응: graveyard/completed에 있는 품목은 in_production을 강제 false 처리 → 중복 방지
4. 매일 사용법 (4단계)
STEP 1
ERP 엑셀 다운로드
ERP에서 4개 파일 받기:
• 라벨조회.xlsx
• 완제품재고창고.xlsx
• 가공작업조회.xlsx
• 발주조회.xlsx
→ ERP_downloaded_data/에 저장
▶
STEP 2
start.bat 더블클릭
refresh.py 자동 실행:
• 엑셀 분석 + JSON 생성
• Google Sheets 업로드
• 브라우저에 대시보드 열림
• 발주 매칭 자동 처리
▶
STEP 3
발주 품목 선택
대시보드에서:
• 카드 클릭 → 생산지시
• [엑셀] 선택 → 발주서 생성
• 실수 시 Undo 가능
• 추적불가 품목 확인/정리
▶
STEP 4
다음날 반복
매일 STEP 1~3 반복하면:
• 발주 건 → 자동 완료
• 재고 찬 건 → 자동 졸업
• 이력은 archived에 보존
• 시스템이 알아서 관리!

6. 자주 묻는 질문
Q. 실수로 카드를 클릭했어요
상단 되돌리기 버튼 또는 "생산지시 중" 카드를 다시 클릭하면 복귀됩니다.
Q. "추적 불가"는 왜 뜨나요?
생산지시 중/완료에 넣었던 품목이 다음 start.bat 실행 시 freq<10이 되어 데이터에서 빠진 경우. 확인 후 [제거]로 정리하세요.
Q. 재고 충분 갔다가 다시 돌아오면?
지시일/완료일 이력이 archived에 보존됩니다. "선생산 필요"에 다시 나타나도 이전 이력은 남아있습니다.
Q. "이미 생산중"은 뭔가요?
7일 내 ERP 가공이력이 있는 품목. 내가 발주한 거면 클릭해서 "생산지시 중"으로 옮기세요.
Q. 인터넷 없이도 되나요?
네! data_wr.js 파일 있으면 HTML 더블클릭 가능. Google Sheets 동기화만 안 됩니다.
Q. graveyard에 넣었는데 "이미 생산중"에도 뜨나요?
아니요. graveyard/completed에 있는 품목은 in_production이 무시되어 중복 표시되지 않습니다.
