📈 ① 트렌드 미니차트

각 품목 카드에 최근 3개월 주문 추이 스파크라인을 표시합니다.
주문이 증가 추세인지, 감소 추세인지 한눈에 파악 가능합니다.
효과: 의사결정 속도 ↑↑
구현 방법: refresh.py에서 주문일자별 그룹핑 → monthly_trend 배열 추가 → 안전재고발주.html에서 Canvas 또는 SVG 스파크라인 렌더링. 외부 라이브러리 없이 SVG path로 구현 가능합니다.
hover하면 구현 방법 표시

⏱️ ② 발주 리드타임 표시

생산지시 → 발주 완료까지 걸린 평균 리드타임을 카드에 표시합니다.
"이 품목은 보통 5일 걸림" → 언제 지시해야 하는지 역산 가능.
효과: 발주 타이밍 최적화
구현 방법: completed 상태의 graveDate↔completedDate 차이를 계산 → 품목별 avg_lead_days 필드 추가. 신규 품목은 전체 평균값으로 대체.
hover하면 구현 방법 표시

📊 ③ 대시보드 통계 강화

상단에 핵심 KPI 카드를 추가합니다:
— 총 선생산 대상 품목 수 / 재고 위험 품목 비율
— 이번 주 완료 건수 / 평균 처리 시간
— 재질별·품종별 분포 도넛 차트
효과: 현황 파악 시간 단축
구현 방법: 안전재고발주.html 상단에 KPI 바 추가. data_wr.json의 기존 summary 데이터(jaejil_summary, pumok_summary) 활용. CSS Grid + SVG 도넛차트로 라이브러리 없이 구현.
hover하면 구현 방법 표시

📂 ④ 미등록 엑셀 2개 연동

🤖 ⑤ 수요 예측 엔진

단순 빈도 카운트를 넘어, 시계열 기반 수요 예측을 도입합니다.

구현 방법: refresh.py에 이동평균(7일/30일) 계산 추가 → predicted_qty, days_of_supply 필드 생성 → 대시보드에 "재고 소진 예상" 게이지 표시. Python pandas 없이 순수 계산으로도 충분.

🔄 ⑥ 자동 Refresh 스케줄러

현재 수동으로 start.bat 실행이 필요합니다.
Windows 작업 스케줄러와 연동하여 매일 아침 자동으로
ERP 데이터 갱신 → JSON 업데이트 → 서버 재시작 처리.
효과: 운영 부담 제거
구현 방법: refresh.py를 독립 실행 가능하게 수정 → Windows Task Scheduler에 매일 08:00 트리거 등록 → server.py는 별도 서비스로 상시 실행. 또는 refresh.py에 watchdog 패턴 추가.
hover하면 구현 방법 표시

👥 ⑩ 다중 사용자 충돌 방지

업그레이드 전후 비교

현재 (AS-IS)

  • 수동으로 start.bat 실행
  • 빈도 기반 단순 스코어링
  • 카드에 숫자만 표시

업그레이드 후 (TO-BE)

  • 매일 자동 갱신 + 알림
  • 시계열 예측 기반 스코어링
  • 트렌드 차트 + KPI 대시보드
  • 리드타임 역산 자동 발주 타이밍
  • 4개 엑셀 전체 연동 완료
  • 의사결정 히스토리 타임라인
  • "N일 후 소진 예상" 게이지

완전 자동화된 선생산 관리 시스템을 목표로 합니다.
